# 缺口追问访谈流程

这是真人简历专家最值钱的环节：通过**主动提问**，把用户脑子里没写进简历的数字、规模、成果挖出来，再如实写回素材库。**只挖掘真实信息，绝不替用户编造**（见 tailoring-rules.md）。

## 何时触发

- 评审 rubric 的「量化」维度 ≤3（见 review-rubric.md）。
- 某条 bullet 只有过程、没有结果/数字。
- JD 的某项硬性要求是「部分」匹配，追问能否补强为「覆盖」。
- 抽取得到的素材库 `metrics` 为空、`skills`/`summary` 缺失。

## 提问原则

1. **一次问 2~4 个**最高价值的问题，不要一次轰炸十几个。
2. 每个问题指向**一条具体 bullet**，并说明为什么问（让用户知道答了有什么用）。
3. 问**可量化**的东西：规模、指标、提升幅度、耗时、用户数、数据量、排名、影响范围。
4. 给**示例答案格式**，降低用户回答成本。
5. 用户答"没有/不知道/没测过" → 尊重，**保持该点不量化**，不追逼、不编造。

## 提问模板（按 bullet 类型）

| bullet 类型 | 追问示例 |
|---|---|
| 做了某系统 | "这个系统服务多少用户/处理多少数据量/QPS？上线了吗？" |
| 做了某优化 | "优化前后的指标变化是多少？（如延迟 X→Y、准确率 +Z%）" |
| 做了某模型 | "离线/在线指标？相比 baseline 提升多少？数据规模多大？" |
| 带了团队/独立做 | "团队几人？你负责哪部分？周期多久？" |
| 拿了奖/排名 | "排名/获奖比例？（如前 5%、1/200）" |

## 回写素材库

用户确认的真实信息**写回 `data/master-profile.yaml`**（经用户确认）：
- 数字 → 写进对应 bullet 的 `metrics`（结构化）。
- 新的真实经历/技能 → 新增条目或 skill，配 `evidence_refs`。
- 这样素材库越用越完整，下次换 JD 直接复用。

## 与裁剪的衔接

访谈在**裁剪之前或之中**进行：先挖到尽量多的真实素材，再据此裁剪改写，最后评审。形成闭环：

```
抽取/加载素材库 → 匹配分析发现薄弱点 → 追问访谈挖数字(写回profile)
   → 裁剪改写(用上新数字) → 评审打分 → 仍弱则再追问 → 交付
```

## 示例对话

> 我注意到「多 Agent 选课推荐系统」这几条很扎实，但缺少能打动 HR 的数字。想问你 3 个问题，补上后这段会强很多：
> 1. 这个系统覆盖/服务了多少学生或多少门课程的数据？
> 2. 推荐的准确率 / 用户满意度 / 召回耗时，有没有测过任一个？
> 3. 这是纯独立项目，还是有线上/课程作业的真实使用场景？
>
> （如果某项没有也没关系，如实告诉我，我不会编造。）
