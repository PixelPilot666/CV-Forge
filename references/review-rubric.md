# 专家评审 Rubric（简历草稿打分）

生成简历后，扮演资深简历专家用本 rubric **逐条**评审，给出分数 + 可执行的改进建议，写入匹配报告的「专家评审」段。目的是把"差不多能用"逼向"专家水准"。

## 六个维度（每项 1~5 分）

| 维度 | 1 分（差） | 5 分（优） |
|---|---|---|
| **影响力 Impact** | 只写职责，没有成果 | 每条都有清晰成果/业务价值，动词强 |
| **量化 Quantification** | 几乎无数字 | 关键 bullet 都有指标/规模/提升幅度 |
| **相关性 Relevance** | 大量与 JD 无关内容 | 内容高度贴合 JD，最相关的在最前 |
| **清晰度 Clarity** | 冗长、术语堆砌、句式混乱 | 简练专业、动词开头、结构统一 |
| **ATS 友好** | 关键词命中少、含图表/多栏 | JD 关键词覆盖好、单栏纯文本 |
| **页面纪律 Layout** | 半页空白 / 溢出 / 孤行 | 接近整页、铺满、无孤行（见 resume-craft A2/A3） |

## 评分与处置

- **总分 ≥ 26/30 且无维度 ≤2**：可交付。
- **任一维度 ≤2**：必须修订该维度后再交付。
- **量化维度 ≤3**：触发缺口追问访谈（见 interview.md），逼出数字。

## 输出格式（写入 match-report.md 的「专家评审」段）

```markdown
## 专家评审（满分 30）

| 维度 | 分 | 说明与改进建议 |
|---|---|---|
| 影响力 | 4 | expe-1-b2 仍偏过程，建议补"上线后…指标变化" |
| 量化 | 3 | proj-agent 各点缺数字：选课系统服务多少学生？召回耗时？→ 触发追问 |
| 相关性 | 5 | 召回-排序主线已前置，贴合 JD |
| 清晰度 | 4 | 个别句偏长，建议拆分 |
| ATS | 4 | 命中良好；可在技能/正文补「向量检索」等 JD 词 |
| 页面纪律 | 5 | 单页铺满，无孤行 |
| **合计** | **25/30** | 修订量化维度后可交付 |

### 逐条改进清单
- [ ] proj-agent-b2：补充服务学生规模 / 推荐准确率（需向用户确认真实数字）
- [ ] expe-1-b2：把"实现个性化推荐"改为带结果的表述
- ...
```

## 评审原则

- 改进建议必须**可执行**（指出具体 bullet + 怎么改），不空泛。
- 凡是建议"补数字/补成果"的，**只能向用户追问真实信息**，绝不替用户编造（见 tailoring-rules.md）。
- 评审不得建议改写 A1 中的 verbatim 字段（公司/岗位/时间/论文名/项目名）。
