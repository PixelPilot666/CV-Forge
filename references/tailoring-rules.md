# 裁剪规则 —— 真实为主，允许合理补强（硬边界）

这是 CV-Forge 最重要的约束。生成简历内容时**必须**遵守。

## 允许（合理补强）

- **重排**经历 / 项目 / bullet 的顺序，把与 JD 最相关的放前面。
- **筛选**：按 JD 相关性选择展示哪些 id，隐藏不相关的（但不得删改事实）。
- **润色**：用 JD 的词汇/术语重写已有 bullet 的**表述**，使其更贴合（语义不变）。
  - 例：profile 写"混合召回"，JD 说"向量检索 + 关键词检索"，可改写为后者的措辞，因为它们指同一件事。
- **突出**已有的**真实** `metrics`：把 profile 里已记录的数字前置/加粗。
- **归组**：按 JD 重新组织技能分组与标题。
- **调整语气与详略**：精简或展开已有内容。

## 禁止（编造）

- 新增任何 profile 中不存在的公司 / 职位 / 时间 / 学历。
- 编造 profile `metrics` 里没有的数字（性能、规模、增长率等）。
- 声称没有 `evidence_refs` 支撑、且 profile 中无对应经历的技能。
- 夸大技能 `level` 或 `years` 超出 profile 记录。
- 把 JD 的关键词写进简历，却在 profile 里找不到任何真实证据支撑。

## 执行机制

1. **可追溯**：每条进入简历的 bullet 必须来自 profile 的某个 bullet `id`（或其润色版本）。`tailor.json` 的 `bullet_ids` / `bullet_overrides` 记录来源，`fill_template.py` 只渲染来自 profile 的文本。
2. **关键词需有证据**：JD 关键词要出现在简历里，必须能在 profile 中解析到对应经历 / 项目 / 技能证据；否则归入**缺口（gap）**，不写入简历。
3. **缺口处理**：对缺口，要么如实留空，要么以问题反问用户（"你是否真的有 X 经验？在哪用过？"），**绝不**用编造填补。
4. **补强审计**：匹配报告中必须包含一段"补强审计"，列出：哪些内容被强调/重排、哪条 bullet 做了措辞润色（标注原 id）、哪些 JD 要求作为缺口未覆盖。让用户一眼看清"强调"与"编造"的边界。

## 一句话

> 我们让简历更好地**呈现**真实的你，而不是**虚构**一个更好的你。
